Список доработок.
1.	Собрал за одну дату минимальный объем json:
a.	meas – это реальные измерения за 2025-10-26 00 мск
b.	seac – это результат оценки режима за тот же момент
c.	df_min, df_max – ограничения, опубликованные на этот момент станцией в Excel
d.	opt1 – 11 точек по мощности, рассчитанные на 2025-10-27 00 мск при отключенных КА-1 и ТА-4, как было фактически в момент расчета 26.10
e.	opt2 – 11 точек по мощности, рассчитанные на 2025-10-27 00 мск при отключенной ТА-4 и включенном КА-1
Процесс расчета мне представляется следующим образом:
•	пользователь запрашивает текущие измерения, получает схему с измерениями
•	нажимает кнопку «Оценить», получает схему с оценкой
•	изменяет измерения, которые его не устраивают, нажимает кнопку «Оценить», пока не получит нормальный режим
o	каждое нажатие кнопки «Оценить» передает в бэкенд json с измененными данными
o	возвращается json с оценкой
•	Фиксирует переменные изменяет состояния, нажимает кнопку «Рассчитать».
o	После этого оцененный режим передается в виде json в бэкенд
o	Ограничения и состояния также передаются в бэкенд. Отключение оборудования не должно менять его оперативные границы, это свойство работающего оборудования.
o	Возвращается json c 11 режимами
•	Для отображения полученных режимов используется новая страница со схемой. Все эти диаграммы и таблицы мало информативны на мой взгляд.
У страницы есть имя режима, которое формируется из включенных турбин с режимами и включенных котлов, например «T1_T2K_T3_K2_K3_K4». Пользователь должен иметь возможность отредактировать название, например, дописать «без РОУ», или «без ТФУ-2», или «лучший!1»
•	Пользователь должен иметь возможность посмотреть режим на схеме, а также – расходную характеристику. Если его все устраивает, то он нажимает «Сохранить», и json с режимами и именем передается в бэкенд для сохранения.
Тут важно иметь в виду, что результаты могут сохраняться на произвольную дату, нужно, чтобы у пользователя была возможность выбрать: на сегодня он считает, на завтра, на 4 дня вперед или на год вперед. В общем, при сохранении, он указывает дату, на которую считает, и она передается в json для сохранения.
•	Пользователь должен видеть, какие режимы уже сохранены, выбрать один из них для отображения. 
•	Если это пользователь со станции, он может захотеть пересчитать режим. Например, взять посчитанный вчера режим, исправить ограничения или просто сохранить его за новую дату.
